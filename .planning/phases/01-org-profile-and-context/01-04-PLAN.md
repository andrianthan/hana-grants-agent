Create `scripts/ingest_documents.py` -- CLI script that reads extracted text + supplementary markdown files, chunks via multi-strategy chunking, embeds via Bedrock Titan, and inserts into the `documents` table with ON CONFLICT upsert.

**CRITICAL: Import and use `EMBEDDING_DIMS` from `utils.config`** -- do NOT hardcode 768 or 1024. This was the exact dimension mismatch the review identified.

Key behavior:
- Accept `--secret-arn` argument (required)
- **Phase 1: Ingest extracted PDF text files from `org-materials/.extracted-text/`**
  - Infer doc_type from parent directory or filename patterns:
    - Files from grant-applications/ or containing "proposal", "application" -> "application"
    - Files from progress-reports/ or containing "report" -> "progress-report"
    - Files from work-plans/ or containing "work_plan", "workplan" -> "work-plan"
    - Other -> "reference"
  - **Metadata derivation for funder and year (addresses review concern):**
    - Parse filename for funder and year using these conventions:
      - Pattern: `FUNDER_YEAR_Description.txt` (e.g., `SAMHSA_2023_Application.txt` -> funder=SAMHSA, year=2023)
      - Pattern: `FYC_CSS_Proposal.txt` -> funder from first token (FYC), year=unknown
      - Year: look for 4-digit pattern matching `20\d{2}` in filename
      - Funder: use first underscore-delimited token, or first word before a year
    - Fallback: `funder="unknown"`, `year=str(current_year)`
- **Phase 2: Ingest supplementary markdown files:**
  - FUNDER-DIRECTORY.md, METRICS-AND-OUTCOMES.md, PARTNER-NETWORK.md, WRITING-STYLE-GUIDE.md -> doc_type="reference", funder="internal", year=current year
  - ORG-PROFILE.md -> doc_type="org_profile", funder="internal", year=current year
  - SEARCH-PROFILES.md -> doc_type="reference", funder="internal", year=current year
- For each file: read text -> chunk_by_section() -> for each chunk: get_embedding() -> INSERT INTO documents ON CONFLICT (source_file, chunk_index) DO UPDATE SET content=EXCLUDED.content, embedding=EXCLUDED.embedding, section_title=EXCLUDED.section_title
- **Retry/backoff for Bedrock calls:** Wrap get_embedding in a retry loop (3 attempts, exponential backoff starting at 1s) to handle transient API failures
- `--fresh` flag truncates documents table before ingesting
- Print progress per file and total chunks at end
- Print summary: files processed, total chunks, chunks per file average, any errors

**Verification step at end:** Query `SELECT COUNT(*) FROM documents` and print the total row count.
