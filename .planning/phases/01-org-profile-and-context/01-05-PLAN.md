<objective>
Generate per-profile HyDE (Hypothetical Document Embedding) queries for all 6 Hanna Center department search profiles. Each profile's HyDE seed prompt is sent to GPT-5.4 to produce a realistic hypothetical grant announcement, which is then embedded via Bedrock Titan V2 and stored in the hyde_queries table. A SHA-256 hash of each profile section enables automatic regeneration when profiles change.

Purpose: The Prospector agent uses HyDE embeddings for vector similarity search -- without them, grant matching cannot run.
Output: `scripts/generate_hyde.py` CLI script that populates the hyde_queries table with 6 HyDE query embeddings.
</objective>

  <action>
Create `scripts/generate_hyde.py` -- CLI script that generates HyDE queries for each search profile and stores them in the hyde_queries table.

Key behavior:
- Accept `--secret-arn` argument (required for DB connection)
- Accept `--profile-id` argument (optional, to regenerate a single profile)
- Accept `--force` flag (regenerate all regardless of hash match)
- Parse org-materials/SEARCH-PROFILES.md to extract profile sections
  - Each profile section starts with `### Profile: \`profile_id\``
  - Extract the full section text between profile headers
  - Extract the HyDE Seed Prompt from the code block within each section
- For each profile:
  1. Compute SHA-256 hash of the profile section text
  2. Check if hyde_queries table already has this profile_id with matching hash
  3. If hash matches and --force not set, skip (profile unchanged)
  4. If hash differs or --force set or profile not in DB:
     a. Send HyDE seed prompt to GPT-5.4 with system prompt instructing it to write a realistic hypothetical grant announcement (500-800 words)
     b. Embed the generated text via get_embedding()
     c. INSERT INTO hyde_queries ON CONFLICT (profile_id) DO UPDATE
- Import HYDE_MODEL from utils.config for the OpenAI model name
- Import EMBEDDING_DIMS from utils.config for dimension verification
- Retry/backoff for both OpenAI and Bedrock calls (3 attempts, exponential backoff)
- Print progress per profile and summary at end
  </action>
